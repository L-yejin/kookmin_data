# 소재 네이밍 컨벤션

## 구조

```
{타입}_{카테고리}_{시즌}_{AB}_{버전}     ← AB 있을 때 (5파트)
{타입}_{카테고리}_{시즌}_{버전}          ← AB 없을 때 (4파트)
```

파트 수(언더스코어 분리)로 AB 유무를 판단한다.

## 소재 타입

| 코드 | 의미 | 집행 채널 |
|---|---|---|
| VID | 영상 | 구글, 메타 |
| IMG | 이미지 | 구글, 메타, 네이버 |
| CRS | 카루셀 | 구글, 메타 |
| BNR | 배너 | TODO: 실제 사용 여부 확인 |

## 카테고리

| 코드 | 내용 |
|---|---|
| 플러스멤버십 | 멤버십 혜택 소구 |
| 적립혜택 | 포인트·적립 혜택 소구 |

| 배송혜택 | 무료배송 등 배송 혜택 소구 |

## AB 구분

| 코드 | 의미 |
|---|---|
| A | 대조군 |
| B | 실험군 |
| (없음) | 단일 소재, AB 테스트 아님 |

## 버전

- `v1`, `v2`, `v3` … 동일 소재의 반복 집행 회차.
- 크리에이티브 내용이 바뀌면 버전 올림. 세팅 변경만은 버전 올리지 않음.

## 파싱 예시

```python
def parse_creative(name: str) -> dict:
    parts = name.split("_")
    if len(parts) == 5:
        return {
            "타입": parts[0], "카테고리": parts[1], "시즌": parts[2],
            "AB": parts[3], "버전": parts[4], "AB여부": True,
        }
    else:  # 4파트
        return {
            "타입": parts[0], "카테고리": parts[1], "시즌": parts[2],
            "AB": None, "버전": parts[3], "AB여부": False,
        }
```
